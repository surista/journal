YouTube Pitch Control Test Suite

Test Overview

This page tests the new YouTube pitch control implementation that allows pitch adjustment independent of playback speed.

How it works:

  1. The implementation uses the Screen Capture API to capture tab audio
  2. Audio is processed through Web Audio API and Tone.js for pitch shifting
  3. Pitch can be adjusted without affecting playback speed
  4. The original YouTube video continues playing at normal speed
⚠️ Important: When you click "Enable Pitch Shifting", your browser will ask you to share tab audio. Make sure to select the current tab for the feature to work properly.

Test Cases

1. Basic Functionality PENDING

Steps:

  1. Load a YouTube video
  2. Click "Enable Pitch Shifting"
  3. Select current tab in the dialog
  4. Verify pitch controls become active

2. Pitch Independence PENDING

Steps:

  1. Enable pitch shifting
  2. Adjust pitch up/down
  3. Verify pitch changes without speed change
  4. Adjust speed separately
  5. Verify speed changes without pitch change

3. Audio Quality PENDING

Check for:

  • No audio dropouts
  • Minimal latency
  • Clean pitch shifting without artifacts
  • Proper stereo imaging

4. UI Behavior PENDING

Verify:

  • Pitch buttons disabled when processor inactive
  • Warning message when trying to use pitch without enabling
  • Proper button state changes
  • Clear visual feedback

5. Edge Cases PENDING

Test:

  • Switching between YouTube and local files
  • Closing tab during audio capture
  • Multiple enable/disable cycles
  • Browser compatibility (Chrome, Edge, Firefox)

Known Limitations

  • Requires user permission for tab audio capture
  • May not work in all browsers (requires getDisplayMedia support)
  • Small latency between video and processed audio
  • Cannot capture audio from muted tabs
